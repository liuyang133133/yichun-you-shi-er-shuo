# P1 Sprint 5 Implementation Plan — SHOULD-9 CAPTCHA

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为短信登录/密码登录/自动注册加 Cloudflare Turnstile 人机验证 + 注册限频（IP 24h / 手机号 7d）+ 新用户 24h 1 帖防灌水。

**Architecture:**
- 新建 `backend/src/modules/captcha/` 模块，包含 `CaptchaService`（provider 抽象）+ `RegisterThrottleService`（IP/手机号/新用户 3 层限频）
- `CaptchaService.isEnabled()` 通过 `CAPTCHA_PROVIDER` 环境变量切换（none=开发, turnstile=生产）
- `AuthService.sendSmsCode / loginBySms / loginByPassword` 入口处调用 `captchaService.verify()`（fail-closed）
- `AuthService.loginBySms` 自动注册场景调用 `registerThrottle.preCheckRegister` + `recordRegister`
- `PostService.create` 入口调用 `registerThrottle.assertCanPost`（新用户 24h 1 帖）
- 集成通过 `AuthModule` re-export `CaptchaModule`，`PostModule` 导入 `AuthModule` 取 `RegisterThrottleService`

**Tech Stack:** NestJS 10, class-validator, ioredis, Cloudflare Turnstile (server-side siteverify via fetch)

## Task 1: CaptchaModule — CaptchaService + RegisterThrottleService

**Files:**
- Create: `backend/src/modules/captcha/captcha.service.ts`
- Create: `backend/src/modules/captcha/register-throttle.service.ts`
- Create: `backend/src/modules/captcha/captcha.module.ts`

### CaptchaService 设计要点

- `isEnabled()`:
  - `CAPTCHA_PROVIDER=none` → false（dev 友好）
  - `CAPTCHA_PROVIDER=turnstile` → true
  - `NODE_ENV=production` 即使配 none 也强制 true（fail-closed 默认）
- `verify(token, ip)`:
  - provider=none → 直接 return
  - provider=turnstile + 无 token → throw 400 请先完成人机验证
  - token 长度 > 2048 → throw 400（防 DoS）
  - Redis 缓存 5 分钟已验证 token（防前端 retry 风暴击穿 turnstile 限频）
  - turnstile siteverify: POST `https://challenges.cloudflare.com/turnstile/v0/siteverify`
    body: `secret=TURNSTILE_SECRET, response=token, remoteip=ip`
    期望 `{success: true}`，否则 throw 400
- 未知 provider → throw 400（fail-closed）
- 日志记录失败原因（error-codes）

### RegisterThrottleService 设计要点

- 阈值常量:
  - `IP_24H_MAX = 5`：同 IP 24h 最多 5 个新注册
  - `PHONE_7D_MAX = 3`：同手机号 7d 最多 3 次注册
  - `NEW_USER_24H_POST_MAX = 1`：新用户（< 24h）最多 1 帖
- `preCheckRegister(ip, phone)`: GET 计数，`>=` 阈值 throw 429
- `recordRegister(ip, phone, userId)`: INCR + EXPIRE，写审计日志
- `assertCanPost(userId)`:
  - 查 user.createdAt，age < 24h 才限制
  - INCR `register:newuser:posts:<uid>:24h`，`>=` 阈值 throw 429

## 关键设计决策

1. **fail-closed 默认**: `NODE_ENV=production` 强制启用 captcha，即使配 none 也视为 turnstile
2. **Token 缓存 5min**: 防前端 retry 风暴击穿 turnstile 限频配额
3. **Provider 抽象**: 当前只实现 `turnstile`，但 service 接口预留扩展（hCaptcha / 极验）
4. **限频分层**:
   - SMS 限频（已存在，sms.service）: 短信成本防护
   - Register 限频（新增）: 抗批量注册
   - 新用户发帖限频（新增）: 抗灌水
5. **依赖注入**: PostModule → AuthModule → CaptchaModule，单向无环
6. **Smoke 测试**: 不依赖 Turnstile 网络（用 fake token 验证 fail-closed 路径 + none 模式验证 happy path）
