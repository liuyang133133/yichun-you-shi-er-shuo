# P1 Sprint 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix 3 high-risk P1 items to close V1.0 production readiness: data consistency (Post 事务), per-request performance (JWT cache), and observability (structured logging).

**Architecture:**
- **T1 (SHOULD-38)** wraps `UserService.findOne` with Redis cache (5-min TTL, `auth:user:<id>` key), invalidate on update/remove.
- **T2 (SHOULD-41)** swaps NestJS default logger for `nestjs-pino` + `pino-http` (auto reqId, structured JSON, zero business code change).
- **T3 (SHOULD-1)** extends `CreatePostDto` to accept optional `detail` field, wraps `PostService.create` with `prisma.$transaction` covering Post + PostHouse/Secondhand/Job/Lifebiz in one round-trip. Old two-HTTP path stays for backward compat.

**Tech Stack:** NestJS 10 + Prisma 5 + ioredis 5 + nestjs-pino 4 + pino-http 10

**Total commits:** ~6-9 (each task = 1-2 commits + reviews), ~5h wall-clock

**Order rationale:**
- T2 first (1.5h, lowest risk, broad observability benefit, no DB schema)
- T1 second (1.5h, narrow blast radius, validates cache invalidation pattern)
- T3 last (2h, most complex — DTO migration + transaction + 4 sub-service touch)

- [ ] **Step 4: Wrap create in $transaction**

Replace the body of `create(userId, dto)` with:

```ts
async create(userId: bigint, dto: CreatePostDto) {
  return this.prisma.$transaction(async (tx) => {
    // 1) 创建主表
    const { detail, ...postData } = dto;
    const post = await tx.post.create({
      data: {
        ...postData,
        userId,
        images: dto.images || [],
      },
    });

    // 2) 如果带 detail,按 type 写对应子表(同事务)
    if (detail) {
      switch (dto.type) {
        case 'house': {
          await tx.postHouse.create({
            data: {
              postId: post.id,
              rentalType: detail.rentalType!,
              propertyType: detail.propertyType!,
              areaSqm: detail.areaSqm!,
              price: detail.price!,
              facilities: detail.facilities as any,
            },
          });
          break;
        }
        case 'secondhand': {
          await tx.postSecondhand.create({
            data: {
              postId: post.id,
              price: detail.price!,
              condition: detail.condition!,
              brand: detail.brand,
              originalPrice: detail.originalPrice,
            },
          });
          break;
        }
        case 'job': {
          // 校验 companyId 属于当前 user(若提供)
          if (detail.companyId) {
            const company = await tx.company.findUnique({
              where: { id: BigInt(detail.companyId) },
            });
            if (!company || company.creatorId !== userId) {
              throw new BadRequestException('公司不存在或不属于当前用户');
            }
          }
          await tx.postJob.create({
            data: {
              postId: post.id,
              salaryMin: detail.salaryMin,
              salaryMax: detail.salaryMax,
              education: detail.education!,
              experience: detail.experience!,
              companyId: detail.companyId ? BigInt(detail.companyId) : null,
            },
          });
          break;
        }
        case 'lifebiz': {
          const validDays = detail.validDays ?? 30;
          const expireAt = new Date(Date.now() + validDays * 86400_000);
          await tx.postLifebiz.create({
            data: {
              postId: post.id,
              price: detail.price,
              contactPhone: detail.contactPhone,
              expireAt,
            },
          });
          break;
        }
        default:
          throw new BadRequestException(`不支持的 post.type: ${dto.type}`);
      }
    }

    return post;
  });
}
```

Import `BadRequestException` from `@nestjs/common` if not already imported.
