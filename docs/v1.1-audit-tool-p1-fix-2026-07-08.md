## 🚨 根因

L3 admin 全审计 (run-2026-07-08T03-11-46) 发现 11 个 FAIL, 其中 **10 个是 dim runner error**:

```
[P1] POST /admin/posts/999999999/audit       Cannot convert undefined or null to object
[P1] POST /admin/posts/999999999/offline     Cannot convert undefined or null to object
[P1] POST /admin/posts/audit-batch           Cannot convert undefined or null to object
[P1] POST /admin/posts/offline-batch         Cannot convert undefined or null to object
[P1] POST /admin/posts/purge                 Cannot convert undefined or null to object
[P1] POST /admin/users/999999999/ban         Cannot convert undefined or null to object
[P1] POST /admin/users/999999999/roles       Cannot convert undefined or null to object
[P1] POST /admin/notifications/broadcast     Cannot convert undefined or null to object
[P1] POST /admin/companies/999999999/verify  Cannot convert undefined or null to object
[P1] POST /admin/companies/999999999/unverify Cannot convert undefined or null to object
```

**根因**: [`_lib/payloads.js`](frontend/tests/e2e/_lib/payloads.js) 里 `INVALID_TYPE` 对象**没有 `forbidNonWhitelisted` 键**, 但 [`api-security-audit.cjs`](frontend/tests/e2e/api-security-audit.cjs:141) `dim2_invalid` 函数引用了它:

```js
const payloads = [
  INVALID_TYPE.stringAsArray,
  INVALID_TYPE.enumOutOfRange,
  INVALID_TYPE.invalidDate,
  INVALID_TYPE.numericFloat,
  INVALID_TYPE.forbidNonWhitelisted,  // ← undefined
];
```

遍历到第 5 个时 `body = undefined`, 接着 L161:

```js
testName: `异常类型 (${Object.keys(body).slice(0, 2).join(',')})`,
//          ^^^^^^^^^^^^^^^^^^^ 抛 "Cannot convert undefined or null to object"
```

**所有 POST/needBody 端点都会跑 dim2**, 所以**所有 needBody=true 的端点**都被炸 — 不限于 admin. 但因为 L1+L2 只跑 admin (P0 修复期间), L3 (高风险深度跑) 才暴露.

## ✅ 修复 (commit)

### 修复 1: `_lib/payloads.js` — 加 `forbidNonWhitelisted` payload

```diff
+ // [V1.1-fix] 维度 13 用了这个名字, 但 payloads.js 之前没定义 → undefined
+ // dim runner 会 Object.keys(undefined) 抛 "Cannot convert undefined or null to object"
+ // 真正测 ValidationPipe 的 forbidNonWhitelisted: 故意传 DTO 不接受的字段
+ forbidNonWhitelisted: { __invalidKey__: 'audit-leak-attempt', admin: true },
```

### 修复 2: `api-security-audit.cjs` — dim2/dim3 加防御

```diff
async function dim2_invalid(method, path, opts) {
  ...
- const payloads = [
+ const payloads = [
    INVALID_TYPE.stringAsArray,
    INVALID_TYPE.enumOutOfRange,
    INVALID_TYPE.invalidDate,
    INVALID_TYPE.numericFloat,
    INVALID_TYPE.forbidNonWhitelisted,
- ];
+ ].filter((b) => b !== undefined && b !== null); // [V1.1-fix] 兜底: 缺失 payload 不能炸
  for (const body of payloads) {
    ...
-   testName: `异常类型 (${Object.keys(body).slice(0, 2).join(',')})`,
+   const keysLabel = (Object.keys(body || {}).slice(0, 2).join(',')) || '(empty)';
+   testName: `异常类型 (${keysLabel})`,
  }
}
```

`dim3_missing` 同样加 `filter` + `body || {}` 兜底.
