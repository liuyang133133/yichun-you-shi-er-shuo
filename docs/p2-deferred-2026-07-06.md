### ✅ 已修 (1 项)

#### P2-06 评论不存在帖 → 404 (原返回 200 + 空)
- **根因**: `CommentService.findByPost()` 直接 `prisma.comment.findMany({ where: { postId } })`, 未校验 post 存在
- **修复**: 加 `prisma.post.findUnique` 校验, 与同模块 `create()` 行为对齐
  - post 不存在 → `NotFoundException` → HTTP 404
  - post 软删 (status=deleted) → 返回空列表 (允许读)
  - post 正常 + 无评论 → 返回空列表 (无变化)
- **文件**: `backend/src/modules/comment/comment.service.ts` (+18 行)
- **单测**: `backend/src/modules/comment/comment.service.spec.ts` 新增 4 例 (404 / 软删 / 正常组装 / 分页) ✅ 4/4 PASS
- **curl 验证**:
  - `GET /posts/99999/comments` → **404** "信息 ID 99999 不存在"
  - `GET /posts/42/comments` (真实 seed post) → 200 + 空数组
  - `GET /posts/50/comments` (高 id 不存在) → 404

#### P2-07 /ai/draft/extract 应 401 (报告采集时 400)

  - B. 前端 fallback 到 `<热门标签云>` + `<最新发布>`

- **决策**: 待 PM 决议 (产品建议接 AI 智能联想) — **延期 V1.1**

#### P2-02 移动端 console 400/404 噪声
- **当前事实**: dev mode Next.js 自动打印 SSR fetch 错误, prod 不打
- **已知源头**:
  - `posts/[postId]/page.tsx:fetchPost()` `catch{}` 吞错, dev console 仍可见
  - `posts/[postId]/post-detail-content.tsx:commentApi.list()` `.catch(()=>{})` 静默
  - `me/page.tsx:authApi.me()` 5xx/网络错误静默
- **决策**: 
  - A. 全部换成 `console.warn` 便于诊断 — 增加噪声
  - B. dev only 控制 + 生产完全静默 — 需细分环境
  - C. 不动, 接受 dev 噪声存在, prod 不受影响
- **倾向**: **C**, dev 噪音是 Next.js 工作机制, 无法彻底消除, 但 prod 用户看不到

#### P2-03 /me/* 个人中心测试覆盖
- **现状**: v1-acceptance 需要登录态, 但登录链路要 SMS, 不在浏览器 E2E
- **候选方案**:
  - A. 后端测试登录态 (NestJS e2e + supertest + 真实 JWT) — 工作量 1-2 天
  - B. 前端单独 Playwright fixture (mock auth) — 不可靠
  - C. 用 admin API 直接造 token 注入 localStorage — 灰盒, 风险
- **倾向**: **A**, 但需要单独的测试安排时间

#### P2-04 Header 4 个 type tab 与首页 hero 重复
- **现状**: 移动端顶导 + 首页 hero 都能进 4 个 type, 信息架构冗余
- **候选方案**:
  - A. 移除 header type tab, 只保留 hero
  - B. 移除 hero, 只保留 header (移动端空间更紧凑)
  - C. 保留双入口, 但加 analytics 看板对照使用率
- **决策**: 待 PM 看 2 周数据后再决 — **延期**

#### P2-05 排序选项缺热度/距离
- **现状**: 仅 latest/oldest/price_asc/price_desc (+ viewCount 修后)
- **候选方案**:
  - **热门 (hot)**: 按 viewCount + favoriteCount × 时效衰减 (后期需 cron 跑分)
  - **距离 (distance)**: 需用 haversine(user.lng, user.lat, post.area.lng/lat), 用户授权位置
- **决策**: 
  - 热门: V1.1 (依赖 P0 时效衰减基础设施)
  - 距离: V2.0 (产品要求位置授权, 隐私合规要设计)
