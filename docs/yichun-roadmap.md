**入口标准**：开发 100% 完工（已满足）+ Phase 1 7 任务完成 + 真实 Admin 业务流跑通。  
**出口标准**：生产 compose 启动 + HTTPS 可访问 + Admin 真实审核流程通 + contactPhone 不再明文。

详见 [yichun-phase1-tasks.md](./yichun-phase1-tasks.md)。

| ID | 任务 | 来源 | 工时 | 优先级 | 阻塞 |
|---|---|---|---|---|---|
| T-P1-01 | MySQL + Redis 密码轮换 | B-1 | 0.5h | 🔴 | 强 |

### 1.2 Phase 1 完成后成果

- ✅ 生产环境（公网 IP）可访问
- ✅ MySQL/Redis 密码非明文
- ✅ 联系人手机号不再明文（个保法合规）
- ✅ Admin 真实跑通：登录 → 看 dashboard → 审核 1 个待审帖 → 状态变 passed
- ✅ LB 探活 / 监控有依据

### 1.3 风险与缓解

| 风险 | 缓解 |
|---|---|
| MySQL 密码轮换导致旧 token 失效 | 同步轮换所有引用（.env、.env.prod、docker-compose）|
| contactPhone 改脱敏影响前端显示 | 前端详情页同步加"联系"按钮 → 登录后才显示完整 |
| Admin e2e 因 SMS 限频跑不通 | 关闭 throttler 临时调试 / 换 IP / 加白名单 |
| 生产 compose 启动后 perf 不达标 | Phase 1 不强求 perf（V1.1 补 OSS 减 DB 压力）|

## 1.5 ⭐ Phase 1.5：V1.0.5 商业化前置（2-4 周）— **2026-06-22 战略纠偏新增**

> **为什么有这一阶段**：V1.0 上线即"骨架完整、血肉缺失"。业务缺口体检发现 3 个致命断层（成交闭环 / 流量入口 / 商业化 0），如不前置修复，平台会变成"又一个没起来的本地分类信息网站"。
> 详细分析：[yichun-business-gaps-2026-06-22.md §0-1](./yichun-business-gaps-2026-06-22.md)

**目标**：在 V1.0 上线后 2-4 周内，补齐 3 大致命断层，让平台从"能跑"变"能成交 + 有流量 + 有收入"。
**入口标准**：Phase 1 完成 + V1.0 真实部署 + 至少 1 周稳定运行。
**出口标准**：
- ✅ IM 端到端跑通，用户能联系到卖家

- ✅ 第一个付费订单产生（哪怕 9.9 元置顶）
- ✅ 第一个 VIP 商家入驻（哪怕 0 元试用）
- ✅ 客服微信群建立，BD 能拉商家

### 1.5.3 关键依赖

- **Phase 1 完成**（V1.0 真实上线）是前置条件
- **T-P15-01 IM 依赖** [PRD.md §3 M07 站内信](./PRD.md)（复用 message 表的 schema，但用 WebSocket 替代 HTTP 轮询）

- **T-P15-02 SEO 不依赖**其他任务，可独立启动

### 1.5.4 风险与缓解

| 风险 | 缓解 |
|---|---|
| WebSocket 服务稳定性 | 用 [Socket.IO](https://socket.io/) 成熟方案 + 客户端断线重连 + 服务端心跳 |
| 支付接入审核严 | 微信支付用 H5（个人订阅号即可），不依赖小程序 |
| 第一个付费订单难 | 先内部员工 / 朋友 / 亲属开 1-2 单当"种子案例"，做截图宣传 |
| SEO 收录慢 | 主动推送 + 站长平台 + 持续 1 个月每天推 |
| Banner 没人买 | 前 1 个月免费给房产中介 / 装修队 / 招聘方挂，攒案例 |
| 商家入驻无动力 | 前 50 个商家 0 元入驻 + 1 个月免费 VIP，攒 50 张"商家主页"截图 |
